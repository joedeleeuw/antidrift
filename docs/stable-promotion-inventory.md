# Stable Promotion Inventory

This is the promotion ledger for the implemented rule set. It is stricter than `ready`: a stable rule needs at least two independent real repositories with non-test-created drift replication, clean controls, zero known false positives, zero known false negatives, no unresolved production concerns, and a grounded Claude advisory review.

This table is derived from `policy/registries/rules.yaml`, `tooling/antidrift/src/policy/chaski-corpus.mjs`, and `tooling/antidrift/src/policy/external-corpus.mjs`. Reduced fixtures do not count.

## Retired From Promotion Work

These rules were removed from the active custom surface and should not consume stable-promotion effort unless new real-code evidence reopens the locked decision.

| Rule                                | Retirement reason                                                                                                                                  |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `antidrift/no-obvious-comment`      | Token-overlap comment restatement was review-style noise, not a deterministic construction pattern.                                                |
| `antidrift/no-role-literal-in-type` | Role literals were too generic without stronger canonical-model context, and no accepted real redeclaration was found.                             |
| `antidrift/no-cast-to-branded`      | The package-specific brand marker had no real adoption or non-test forgery evidence.                                                               |
| `antidrift/no-unsafe-cast-chain`    | Double-cast tunnels are covered by `@typescript-eslint/no-unsafe-type-assertion`; `no-appeasement-cast` owns the source-boundary authority policy. |

| `antidrift/no-raw-tailwind-color`   | Removed from the shipped plugin surface because the class-token sampler hardcoded a tiny palette instead of proving semantic token authority.       |
| `antidrift/no-hover-translate-card` | Removed from the shipped plugin surface because the hover-translate sampler matched class strings without proving card or pointer-target context.   |
