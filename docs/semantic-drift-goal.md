# Semantic Drift Goal

Antidrift investigates semantic drift in agent-written code by recovering associations that source text alone often hides: state cells to lifecycle roles, parsed values to schemas, local types to generated, domain, or package owners, SQL fragments to binding or escaping proof, route params to authorization boundaries, and broad inputs to validated contracts. The primary path is deterministic semantic adapters over TypeScript, ESLint, registries, and repo graphs. Model-assisted approaches such as LoRA adapters, embeddings, or code-index adapters are exploration tracks for discovery, clustering, and owner proposal, not enforcement dependencies unless they prove measurable value against real code.

Validation is corpus-first. Real codebase copies are the proving ground for drift, clean controls, owner discovery, and false-positive pressure; fixtures are only regression harnesses after behavior has been observed in real code. Each promoted rule must declare its proof bucket, semantic association or authority fact, blocking threshold, ecosystem comparison, corpus evidence, and no-sink or no-dead-work behavior. If a rule cannot prove its broken association deterministically, it remains inventory-only, delegated, research, or retired.

Agent-ops is the minimized fallback proof surface, not the default destination for hard rules. A rule belongs in agent-ops only when it depends on session facts that code semantics cannot recover: task intent, command history, skipped verification, tool precedence, diff sprawl, or generated artifact handling. Completion means Antidrift is a distributable package with a defensible active rule set, shared semantic adapters, corpus-backed promotion gates, multiple research tracks for discovery, and clear retirement or delegation paths for anything that cannot meet the semantic proof bar.

The operational rule-by-rule migration and validation checklist lives in `docs/semantic-validation-matrix.md`.
